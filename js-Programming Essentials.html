Vì sao nên học JavaScript?

JavaScript (JS) là "ngôn ngữ của web" – thứ khiến các trang web trở nên sống động, biến những đoạn HTML tĩnh và CSS đơn giản thành những trải nghiệm tương tác: nút bấm phản hồi, biểu mẫu thông minh, hiệu ứng chuyển động mượt mà, hay thậm chí là các ứng dụng web đầy đủ chức năng như Gmail hoặc Facebook.

Với JavaScript, bạn có thể làm được cả front-end (giao diện người dùng) lẫn back-end (máy chủ qua Node.js), tạo nên những ứng dụng thực sự hoàn chỉnh. Điểm đặc biệt của JS là bạn thấy kết quả ngay lập tức – chỉ cần lưu, làm mới, và thay đổi xuất hiện ngay. Điều này giúp người mới học cảm nhận được sự tiến bộ rõ ràng từng ngày.

Ngoài ra, việc học JavaScript giúp bạn hình thành tư duy lập trình: hiểu cách sự kiện diễn ra, xử lý bất đồng bộ, chia nhỏ chương trình thành các hàm, và gỡ lỗi hiệu quả. Thành thạo JS sẽ giúp bạn tiếp cận các framework hiện đại như React, Vue, hay Svelte dễ dàng hơn.

Tóm lại, JavaScript là một ngôn ngữ linh hoạt, dễ học, và cực kỳ hữu ích — cánh cửa mở ra thế giới lập trình web đầy sáng tạo.

Front-End

Tạo giao diện người dùng tương tác với HTML, CSS và JavaScript

Back-End

Xây dựng server với Node.js và các framework như Express

Mobile Apps

Phát triển ứng dụng di động với React Native hoặc Ionic

Desktop Apps

Tạo ứng dụng desktop với Electron hoặc NW.js

Các khái niệm cốt lõi

Biến (Variables)

Biến là "hộp chứa giá trị". JavaScript cung cấp ba cách khai báo:

const – Dùng cho giá trị cố định, không bị thay đổi.
let – Dùng khi cần thay đổi giá trị.
var – Cách cũ, có phạm vi theo hàm. Hạn chế sử dụng trong code mới. 

// Khai báo biến
const PI = 3.14159;  // Không thể thay đổi giá trị
let count = 0;       // Có thể thay đổi giá trị
var oldWay = "Avoid"; // Cách cũ, không nên dùng

Kiểu dữ liệu (Data Types)

JavaScript có hai nhóm chính:

1️⃣ Kiểu nguyên thủy (Primitive types): Number, String, Boolean, Null, Undefined, Symbol.
2️⃣ Kiểu tham chiếu (Reference types): Object, Array, Function. 

// Primitive types
const age = 25;           // Number
const name = "John";      // String
const isStudent = true;   // Boolean
const nothing = null;     // Null
let undefinedVar;         // Undefined

// Reference types
const person = {          // Object
  name: "John",
  age: 25
};
const colors = ["red", "green", "blue"];  // Array
const greet = function() {                // Function
  console.log("Hello!");
};

⚠️ JS là ngôn ngữ "dynamically typed" — nghĩa là một biến có thể thay đổi kiểu giá trị trong lúc chạy. Và nhớ nhé, typeof null trả về "object" — lỗi lịch sử nhỏ nhưng nổi tiếng.

Hàm (Functions)

Hàm giúp bạn gom nhóm các đoạn mã có cùng mục đích, giúp code dễ đọc và tái sử dụng. Dưới đây là ba kiểu hàm phổ biến:

1. Function Declaration: Cách khai báo truyền thống, dễ đọc và rõ ràng.

function calculateArea(width, height) {
  return width * height;
}

2. Function Expression: Gán một hàm vào biến — linh hoạt khi viết hàm ẩn danh.

const calculateArea = function(width, height) {
  return width * height;
};

3. Arrow Function: Cú pháp ngắn gọn, được dùng nhiều trong ES6.

const calculateArea = (width, height) => {
  return width * height;
};

// Hoặc ngắn gọn hơn với một biểu thức
const calculateArea = (width, height) => width * height;

Arrow function cực kỳ tiện khi viết callback hoặc xử lý ngắn gọn, nhưng hãy nhớ: nó không có từ khóa this riêng — điều này ảnh hưởng khi làm việc với object hoặc class.

Phạm vi (Scope)

"Scope" quyết định biến có thể được truy cập ở đâu:

🔸 Global scope: Biến khai báo ngoài hàm, dùng được ở mọi nơi.
🔸 Local scope: Biến chỉ tồn tại trong hàm.
🔸 Block scope: Với let và const, biến chỉ tồn tại trong cặp ngoặc { }. 

// Global scope
const globalVar = "I'm global";

function example() {
  // Local scope
  const localVar = "I'm local";
  
  if (true) {
    // Block scope
    let blockVar = "I'm block scoped";
    console.log(blockVar); // Works
  }
  
  console.log(blockVar); // Error: blockVar is not defined
}

Hãy ưu tiên let và const để tránh lỗi "rò rỉ biến" không mong muốn.

DOM & Sự kiện – làm trang web sống động

DOM (Document Object Model) là cách JavaScript "nhìn thấy" trang HTML của bạn. Nhờ DOM, JS có thể đọc nội dung, thay đổi phần tử, phản hồi thao tác người dùng, và tạo trải nghiệm tương tác.

Truy xuất phần tử

  • document.getElementById('id')
  • document.querySelector('.class') – chọn phần tử đầu tiên khớp.
  • document.querySelectorAll('div') – chọn tất cả phần tử cùng loại.

Cập nhật DOM

  • element.textContent – thay đổi nội dung văn bản.
  • element.classList.add('active') – thay đổi lớp CSS.
  • parent.appendChild(child) – thêm phần tử mới vào trang.

Lắng nghe sự kiện

const btn = document.querySelector('#myBtn');
btn.addEventListener('click', () => {
  alert('Bạn vừa bấm nút!');
});

Dùng addEventListener thay vì onclick để có thể gắn nhiều hành vi vào cùng một sự kiện.

ES6+ – viết JavaScript hiện đại

ES6 (2015) mang lại hàng loạt cải tiến khiến JS dễ đọc, mạnh mẽ hơn:

let & const

Giúp kiểm soát biến rõ ràng và an toàn hơn so với var.

Arrow function

const square = x => x * x;

Template literals

const name = 'Hieu';
console.log(`Xin chào, ${name}!`);

Destructuring

const user = { name: 'Linh', age: 23 };
const { name, age } = user;

Modules

// utils.js
export const sum = (a, b) => a + b;

// main.js
import { sum } from './utils.js';

Async/Await

async function fetchData() {
  const res = await fetch('/api/data');
  const data = await res.json();
  console.log(data);
}

Ví dụ nhỏ – "Xin chào từ JavaScript"

Đặt đoạn mã này vào cuối HTML của bạn và xem kết quả: 

const btn = document.querySelector('#runBtn');
const output = document.querySelector('#output');
btn.addEventListener('click', () => {
  const name = prompt('Nhập tên của bạn:') || 'bạn';
  output.textContent = `Xin chào, ${name}! Chúc bạn học JS vui vẻ 🎉`;
});

Lộ trình luyện tập

  • 2–3 tuần đầu: biến, kiểu dữ liệu, vòng lặp, hàm, mảng, đối tượng.
  • 1–2 tuần tiếp: DOM, sự kiện, thao tác giao diện.
  • 2 tuần: ES6+, modules, arrow functions, template strings.
  • 2 tuần sau: Async/Await, fetch API, xử lý lỗi.

Hãy luyện tập qua các mini project như: to-do list, quiz app, đồng hồ kỹ thuật số, hay trò chơi đoán số. Càng làm nhiều, bạn càng "ngấm" JavaScript tự nhiên hơn.

Khi đã nắm vững nền tảng, hãy tiến lên học React hoặc Vue – đó là lúc JavaScript thật sự phát huy sức mạnh.
