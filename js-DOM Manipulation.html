Làm Chủ JavaScript DOM Manipulation: Hướng Dẫn Toàn Diện

19 tháng 10, 2024 20 phút đọc JavaScript, DOM, Frontend

Giới Thiệu

DOM (Document Object Model) là giao diện lập trình (API) cho phép JavaScript tương tác với nội dung, cấu trúc và style của trang web. Hiểu và làm chủ DOM Manipulation là kỹ năng quan trọng nhất của mọi Frontend Developer.

Trong bài viết này, chúng ta sẽ khám phá từ những khái niệm cơ bản nhất đến các kỹ thuật nâng cao, kèm theo ví dụ thực tế và best practices.

1. DOM Là Gì Và Tại Sao Quan Trọng?

1.1. Khái Niệm Cơ Bản

Khi trình duyệt tải một trang HTML, nó phân tích cấu trúc và tạo ra một cây DOM - một biểu diễn dạng cây của toàn bộ tài liệu HTML. Mỗi phần tử HTML trở thành một nút (node) trong cây này.

<!DOCTYPE html> <html> <head> <title>My Page</title> </head> <body> <h1 id="title">Hello World</h1> <div class="container"> <p>This is a paragraph</p> </div> </body> </html>

Cấu trúc DOM sẽ là: Document → html → (head, body) → body chứa h1 và div → div chứa p

1.2. Các Loại Node Quan Trọng

  • Element Node: Các thẻ HTML (div, p, h1, ...)
  • Text Node: Văn bản bên trong các element
  • Attribute Node: Các thuộc tính của element (class, id, src, ...)
  • Comment Node: Các comment trong HTML

2. Truy Cập Phần Tử DOM

2.1. Phương Thức Cổ Điển

// Truy cập theo ID - trả về một phần tử const titleElement = document.getElementById('title'); // Truy cập theo class - trả về HTMLCollection (live collection) const items = document.getElementsByClassName('item'); // Truy cập theo tag name - trả về HTMLCollection const paragraphs = document.getElementsByTagName('p'); // Truy cập theo name attribute const forms = document.getElementsByName('username');

2.2. Phương Thức Hiện Đại (Khuyến Nghị)

// querySelector - trả về phần tử đầu tiên khớp với CSS selector const button = document.querySelector('#submitBtn'); const firstItem = document.querySelector('.item'); const firstParagraph = document.querySelector('p'); // querySelectorAll - trả về NodeList (static collection) const allButtons = document.querySelectorAll('.btn'); const allParagraphs = document.querySelectorAll('p'); // Sử dụng CSS selectors phức tạp const activeItems = document.querySelectorAll('.item.active'); const directChildren = document.querySelectorAll('div > p'); const siblings = document.querySelectorAll('h1 + p');

Mẹo Quan Trọng

Luôn ưu tiên sử dụng querySelector và querySelectorAll vì:

  • Hỗ trợ tất cả CSS selectors
  • Trả về NodeList (có thể sử dụng forEach)
  • Linh hoạt và dễ đọc hơn
  • Không phải là live collection (hiệu suất tốt hơn)

3. Thay Đổi Nội Dung & Thuộc Tính

3.1. Thao Tác Với Nội Dung

const element = document.querySelector('#myElement'); // Thay đổi nội dung văn bản element.textContent = 'Nội dung mới'; // Chỉ lấy text, an toàn hơn element.innerText = 'Nội dung mới'; // Tính đến CSS, hiệu suất kém hơn element.innerHTML = '<strong>Nội dung HTML</strong>'; // Cho phép HTML // Thao tác với thuộc tính element.setAttribute('data-custom', 'value'); // Thêm attribute const value = element.getAttribute('data-custom'); // Lấy giá trị attribute element.removeAttribute('data-custom'); // Xóa attribute // Thuộc tính trực tiếp element.id = 'newId'; element.className = 'new-class'; element.href = 'https://example.com';

3.2. Làm Việc Với CSS Classes

const box = document.querySelector('.box'); // classList - phương thức hiện đại box.classList.add('active', 'highlight'); // Thêm nhiều class box.classList.remove('hidden'); // Xóa class box.classList.toggle('dark-mode'); // Bật/tắt class box.classList.replace('old', 'new'); // Thay thế class box.classList.contains('active'); // Kiểm tra class tồn tại // Thao tác style trực tiếp box.style.backgroundColor = 'blue'; box.style.fontSize = '20px'; box.style.display = 'none';

4. Tạo & Quản Lý Phần Tử Mới

4.1. Tạo Và Chèn Phần Tử

// Tạo phần tử mới const newDiv = document.createElement('div'); newDiv.textContent = 'Đây là div mới'; newDiv.className = 'custom-div'; // Các phương thức chèn const container = document.querySelector('#container'); container.appendChild(newDiv); // Thêm vào cuối container.prepend(newDiv); // Thêm vào đầu container.insertBefore(newDiv, reference); // Chèn trước phần tử khác // Chèn HTML trực tiếp container.insertAdjacentHTML('beforeend', '<p>Đoạn văn mới</p>'); // Các vị trí insertAdjacentHTML // 'beforebegin': Trước element // 'afterbegin': Bên trong, ở đầu // 'beforeend': Bên trong, ở cuối // 'afterend': Sau element

4.2. Xóa Và Thay Thế Phần Tử

const element = document.querySelector('#toRemove'); const newElement = document.createElement('span'); // Xóa phần tử element.remove(); // Phương thức hiện đại element.parentNode.removeChild(element); // Phương thức cũ // Thay thế phần tử element.replaceWith(newElement); // Phương thức hiện đại element.parentNode.replaceChild(newElement, element); // Phương thức cũ // Xóa tất cả con container.innerHTML = ''; // Nhanh nhưng không an toàn while (container.firstChild) { container.removeChild(container.firstChild); // An toàn hơn }

5. Xử Lý Sự Kiện (Event Handling)

5.1. Các Loại Sự Kiện Phổ Biến

const button = document.querySelector('#myButton'); // Thêm event listener button.addEventListener('click', function(event) { event.preventDefault(); // Ngăn hành vi mặc định console.log('Button được click!'); console.log('Target:', event.target); console.log('Current Target:', event.currentTarget); }); // Các sự kiện thông dụng element.addEventListener('mouseover', handleMouseOver); element.addEventListener('mouseout', handleMouseOut); element.addEventListener('focus', handleFocus); element.addEventListener('blur', handleBlur); element.addEventListener('keydown', handleKeyDown); element.addEventListener('submit', handleSubmit);

5.2. Event Delegation - Kỹ Thuật Quan Trọng

// THÔNG THƯỜNG - Không hiệu quả const items = document.querySelectorAll('.item'); items.forEach(item => { item.addEventListener('click', () => { console.log('Item clicked'); }); }); // EVENT DELEGATION - Hiệu quả const list = document.querySelector('#itemList'); list.addEventListener('click', (event) => { if (event.target.classList.contains('item')) { console.log('Item clicked:', event.target.textContent); } });

Tại Sao Event Delegation Quan Trọng?

  • Hiệu suất: Chỉ cần một event listener thay vì nhiều
  • Dynamic Content: Hoạt động với phần tử được thêm sau
  • Bộ nhớ: Tiết kiệm bộ nhớ với danh sách lớn

6. Kỹ Thuật Nâng Cao & Best Practices

6.1. Data Attributes & Dataset

// HTML: <div data-user-id="123" data-role="admin"></div> const userDiv = document.querySelector('[data-user-id]'); // Truy cập data attributes const userId = userDiv.dataset.userId; // "123" const userRole = userDiv.dataset.role; // "admin" // Thay đổi data attributes userDiv.dataset.userId = '456'; userDiv.dataset.status = 'active';

6.2. DocumentFragment - Tối Ưu Hiệu Suất

// Tạo nhiều phần tử cùng lúc const fragment = document.createDocumentFragment(); for (let i = 0; i < 1000; i++) { const li = document.createElement('li'); li.textContent = `Item ${i}`; fragment.appendChild(li); } // Chèn một lần duy nhất document.querySelector('#list').appendChild(fragment);

6.3. MutationObserver - Theo Dõi Thay Đổi DOM

const observer = new MutationObserver((mutations) => { mutations.forEach((mutation) => { if (mutation.type === 'childList') { console.log('Có phần tử được thêm/xóa'); } }); }); observer.observe(document.querySelector('#container'), { childList: true, subtree: true });

7. Ví Dụ Thực Tế: Ứng Dụng Todo List

class TodoApp { constructor() { this.tasks = JSON.parse(localStorage.getItem('tasks')) || []; this.initializeApp(); } initializeApp() { this.form = document.querySelector('#todoForm'); this.input = document.querySelector('#taskInput'); this.list = document.querySelector('#todoList'); this.form.addEventListener('submit', (e) => this.addTask(e)); this.list.addEventListener('click', (e) => this.handleTaskClick(e)); this.renderTasks(); } addTask(e) { e.preventDefault(); const taskText = this.input.value.trim(); if (taskText) { const task = { id: Date.now(), text: taskText, completed: false, createdAt: new Date().toISOString() }; this.tasks.push(task); this.saveTasks(); this.renderTasks(); this.input.value = ''; } } handleTaskClick(e) { const taskElement = e.target.closest('.task'); if (!taskElement) return; const taskId = parseInt(taskElement.dataset.id); if (e.target.classList.contains('delete-btn')) { this.deleteTask(taskId); } else if (e.target.classList.contains('complete-btn')) { this.toggleComplete(taskId); } } deleteTask(id) { this.tasks = this.tasks.filter(task => task.id !== id); this.saveTasks(); this.renderTasks(); } toggleComplete(id) { this.tasks = this.tasks.map(task => task.id === id ? { ...task, completed: !task.completed } : task ); this.saveTasks(); this.renderTasks(); } renderTasks() { this.list.innerHTML = ''; const fragment = document.createDocumentFragment(); this.tasks.forEach(task => { const li = document.createElement('li'); li.className = `task ${task.completed ? 'completed' : ''}`; li.dataset.id = task.id; li.innerHTML = ` ${task.text}
`; fragment.appendChild(li); }); this.list.appendChild(fragment); } saveTasks() { localStorage.setItem('tasks', JSON.stringify(this.tasks)); } } // Khởi chạy ứng dụng document.addEventListener('DOMContentLoaded', () => { new TodoApp(); });

8. Tối Ưu Hiệu Suất DOM

Best Practices

  • Tránh Forced Synchronous Layout: Đọc và ghi DOM riêng biệt
  • Sử dụng DocumentFragment: Khi thêm nhiều phần tử cùng lúc
  • Event Delegation: Giảm số lượng event listeners
  • Debounce Input Events: Với search, resize, scroll
  • Virtual DOM: Hiểu nguyên lý để học React/Vue sau này

Kết Luận

DOM Manipulation là nền tảng của mọi ứng dụng web hiện đại. Bằng cách làm chủ các kỹ thuật trong bài viết này, bạn đã trang bị cho mình công cụ mạnh mẽ để xây dựng những trải nghiệm web phong phú và tương tác.

Hãy thực hành thường xuyên, bắt đầu từ những dự án nhỏ và dần dần áp dụng các kỹ thuật nâng cao. Mỗi dòng code bạn viết hôm nay là bước đệm cho những ứng dụng phức tạp trong tương lai.
