1. Giới thiệu về Exception Handling

📖 Tại sao cần Exception Handling?

Trong quá trình lập trình Java, lỗi (error) và ngoại lệ (exception) là điều không thể tránh khỏi. Java Exception Handling giúp bạn kiểm soát luồng chương trình khi có sự cố xảy ra — thay vì để ứng dụng bị dừng đột ngột.

Việc hiểu rõ cơ chế xử lý ngoại lệ, cách tạo custom exceptions và áp dụng best practices sẽ giúp code của bạn trở nên ổn định, dễ bảo trì và thân thiện với người dùng hơn.

Lợi ích của Exception Handling

  • Tách biệt code xử lý lỗi khỏi code logic chính
  • Ngăn chặn crash ứng dụng không mong muốn
  • Cung cấp thông tin lỗi hữu ích cho debug
  • Bảo vệ tài nguyên khỏi bị leak
  • Tăng tính ổn định và độ tin cậy của ứng dụng

2. Phân loại Exception trong Java

Java chia exceptions thành 3 loại chính dựa trên phân cấp kế thừa:

Cấu trúc phân cấp Exception trong Java

Phân cấp Exception trong Java - Màu đỏ: Error, Xanh lá: Checked Exception, Xanh nhạt: Unchecked Exception

So sánh Checked vs Unchecked Exceptions

Tiêu chí Checked Exceptions Unchecked Exceptions
Thời điểm kiểm tra Compile-time Runtime
Bắt buộc xử lý Có (phải catch hoặc declare) Không (tùy chọn)
Kế thừa từ Exception (không phải RuntimeException) RuntimeException
Ví dụ IOException, SQLException NullPointerException, ArrayIndexOutOfBoundsException
Mục đích Lỗi có thể phục hồi Lỗi lập trình

"Hiểu rõ phân cấp của Exception là bước đầu để kiểm soát lỗi một cách thông minh."

3. Cấu trúc try–catch–finally

Java cung cấp khối try–catch để bắt và xử lý ngoại lệ. Ngoài ra, khối finally luôn được thực thi — kể cả khi có lỗi hay không — giúp bạn giải phóng tài nguyên như kết nối file, database, hoặc stream.

Cơ bản
try {
    // Code có thể gây exception
    int result = 10 / 0;
} catch (ArithmeticException e) {
    // Xử lý exception
    System.out.println("Lỗi: " + e.getMessage());
}
Với Finally
try {
    // Code có thể gây exception
    int result = 10 / 0;
} catch (ArithmeticException e) {
    System.out.println("Lỗi: " + e.getMessage());
} finally {
    // Luôn được thực thi
    System.out.println("Khối finally");
}

💡 Lời Khuyên Thực Tế

Hãy luôn dùng finally (hoặc try-with-resources) để đảm bảo tài nguyên được đóng đúng cách.

4. Tạo Custom Exception

Đôi khi, bạn cần định nghĩa lỗi riêng phù hợp với logic nghiệp vụ của mình. Khi đó, có thể tạo Custom Exception bằng cách kế thừa lớp Exception hoặc RuntimeException.

Checked Custom Exception
class InvalidAgeException extends Exception {
    public InvalidAgeException(String message) {
        super(message);
    }
    
    public InvalidAgeException(String message, Throwable cause) {
        super(message, cause);
    }
}
Unchecked Custom Exception
class BusinessRuleException extends RuntimeException {
    private String errorCode;
    
    public BusinessRuleException(String message, String errorCode) {
        super(message);
        this.errorCode = errorCode;
    }
    
    public String getErrorCode() {
        return errorCode;
    }
}

Khi nào nên tạo Custom Exception?

  • Khi bạn cần nhóm nhiều lỗi liên quan thành một loại
  • Khi cần thêm thông tin bổ sung (error code, timestamp)
  • Khi muốn ẩn chi tiết triển khai từ người dùng API
  • Khi cần phân loại lỗi nghiệp vụ cụ thể

5. Multi-Catch & Nested Try

Khi có nhiều loại lỗi có thể xảy ra, bạn có thể dùng multi-catch hoặc nested try để xử lý chi tiết từng trường hợp.

Multi-Catch (Java 7+)
try {
    // Code có thể gây nhiều exceptions
    String text = null;
    System.out.println(text.length());
    int result = 10 / 0;
} catch (ArithmeticException | NullPointerException e) {
    System.out.println("Lỗi: " + e.getClass().getSimpleName());
}
Nested Try-Catch
try {
    // Outer try
    try {
        // Inner try - xử lý chi tiết
        int[] arr = new int[5];
        arr[10] = 100; // ArrayIndexOutOfBounds
    } catch (ArrayIndexOutOfBoundsException e) {
        System.out.println("Lỗi index: " + e.getMessage());
    }
} catch (Exception e) {
    System.out.println("Lỗi tổng quát: " + e.getMessage());
}

6. Throw vs Throws

Throw - Ném exception
public void validateAge(int age) {
    if (age < 0) {
        throw new IllegalArgumentException(
            "Tuổi không thể âm: " + age
        );
    }
    if (age < 18) {
        throw new ArithmeticException(
            "Chưa đủ tuổi: " + age
        );
    }
}
Throws - Khai báo exception
public void readFile(String filename) 
    throws FileNotFoundException, IOException {
    
    BufferedReader reader = new BufferedReader(
        new FileReader(filename)
    );
    // Đọc file...
    reader.close();
}

Quy tắc sử dụng Throw và Throws

  • Throw: Dùng trong thân phương thức để ném exception cụ thể
  • Throws: Dùng trong khai báo phương thức để chỉ ra các exceptions có thể được ném
  • Một phương thức có thể có cả throw và throws
  • Với unchecked exceptions, throws là tùy chọn

"Sử dụng throw và throws hợp lý giúp bạn quản lý lỗi có kiểm soát và rõ ràng."

7. Try-with-resources (Java 7+)

try-with-resources tự động đóng các tài nguyên implement AutoCloseable, giúp code gọn hơn và tránh rò rỉ bộ nhớ.

Try-with-resources
try (BufferedReader reader = new BufferedReader(
        new FileReader("data.txt"))) {
    
    String line;
    while ((line = reader.readLine()) != null) {
        System.out.println(line);
    }
} catch (IOException e) {
    System.out.println("Lỗi đọc file: " + e.getMessage());
}
Nhiều resources
try (FileInputStream input = new FileInputStream("in.txt");
     FileOutputStream output = new FileOutputStream("out.txt")) {
    
    byte[] buffer = new byte[1024];
    int bytesRead;
    while ((bytesRead = input.read(buffer)) != -1) {
        output.write(buffer, 0, bytesRead);
    }
}

8. Exception Propagation

Khi một exception không được bắt trong phương thức, nó sẽ được propagate (lan truyền) lên phương thức gọi nó, và tiếp tục cho đến khi được bắt hoặc chương trình kết thúc. 

public class PropagationExample {
    void methodA() {
        methodB();  // Exception từ methodB được propagate lên đây
    }
    
    void methodB() {
        methodC();  // Exception từ methodC được propagate lên đây
    }
    
    void methodC() {
        throw new RuntimeException("Lỗi từ methodC");
    }
    
    public static void main(String[] args) {
        try {
            new PropagationExample().methodA();
        } catch (RuntimeException e) {
            System.out.println("Bắt được exception: " + e.getMessage());
        }
    }
}

Call Stack Trace

Khi exception xảy ra, JVM tạo một stack trace cho thấy đường đi của exception qua các phương thức. Đây là công cụ debug cực kỳ hữu ích.

9. Best Practices khi xử lý Exception

🚫 Chỉ bắt những exception mà bạn có thể xử lý hợp lý

Không nên bắt tất cả exceptions mà không có xử lý phù hợp.

// KHÔNG NÊN
try {
    // code
} catch (Exception e) {
    // Bắt mọi exception mà không xử lý
}

// NÊN
try {
    // code
} catch (FileNotFoundException e) {
    // Xử lý cụ thể: tạo file mới
    createNewFile();
} catch (IOException e) {
    // Xử lý cụ thể: log và thông báo
    logger.error("Lỗi IO", e);
    showErrorToUser("Không thể đọc file");
}

📝 Luôn ghi log chi tiết

Sử dụng logging framework để ghi lại thông tin lỗi đầy đủ.

try {
    // business logic
} catch (BusinessException e) {
    logger.error("Lỗi nghiệp vụ - Mã lỗi: {}, Ngữ cảnh: {}", 
        e.getErrorCode(), context, e);
    throw new UserFriendlyException("Có lỗi xảy ra, vui lòng thử lại");
}

🧹 Dọn dẹp tài nguyên với try-with-resources

Luôn sử dụng finally hoặc try-with-resources để đảm bảo tài nguyên được giải phóng.

🎯 Cung cấp thông tin lỗi hữu ích

Exception messages nên rõ ràng, giải thích được nguyên nhân và gợi ý giải pháp.

// Thông tin kém
throw new Exception("Lỗi");

// Thông tin tốt
throw new IllegalArgumentException(
    "Giá trị không hợp lệ: " + value + 
    ". Phải nằm trong khoảng " + min + "-" + max
);

10. Common Exceptions & Solutions

Exception Nguyên nhân Giải pháp
NullPointerException Truy cập phương thức/trường của object null Kiểm tra null trước khi sử dụng, sử dụng Optional
ArrayIndexOutOfBoundsException Truy cập index ngoài phạm vi mảng Kiểm tra index trước khi truy cập, sử dụng for-each loop
ClassCastException Ép kiểu không hợp lệ Kiểm tra kiểu với instanceof trước khi cast
IllegalArgumentException Tham số phương thức không hợp lệ Validate input parameters đầu phương thức
IOException Lỗi input/output operations Xử lý đóng tài nguyên, check file existence

11. Chủ đề nâng cao

Exception Translation

Chuyển đổi low-level exceptions thành high-level exceptions phù hợp với abstraction level. 

public class UserService {
    public User findUserById(String id) throws UserNotFoundException {
        try {
            return userRepository.findById(id);
        } catch (SQLException e) {
            // Chuyển SQLException thành UserNotFoundException
            throw new UserNotFoundException("Không tìm thấy user: " + id, e);
        }
    }
}

Exception Chaining

Giữ nguyên nguyên nhân gốc (cause) khi throw exception mới. 

try {
    // code có thể gây IOException
} catch (IOException e) {
    throw new BusinessException("Lỗi xử lý dữ liệu", e);
}

Custom Exception với Additional Data

public class ValidationException extends RuntimeException {
    private String fieldName;
    private Object invalidValue;
    private String validationRule;
    
    public ValidationException(String fieldName, Object invalidValue, 
                              String validationRule, String message) {
        super(message);
        this.fieldName = fieldName;
        this.invalidValue = invalidValue;
        this.validationRule = validationRule;
    }
    
    // Getters
    public String getFieldName() { return fieldName; }
    public Object getInvalidValue() { return invalidValue; }
    public String getValidationRule() { return validationRule; }
}

13. Tổng kết

Exception Handling là kỹ năng quan trọng giúp bạn viết ứng dụng Java an toàn và đáng tin cậy hơn. Biết cách sử dụng try-catch, throw, throws và custom exception đúng lúc sẽ giúp bạn kiểm soát luồng lỗi linh hoạt.

👉 Hãy luyện tập bằng cách cố tình tạo lỗi trong chương trình nhỏ, sau đó xử lý chúng theo nhiều cách khác nhau — bạn sẽ nhanh chóng hiểu sâu hơn về cơ chế Exception trong Java.

  • Hiểu rõ phân cấp Exception và sự khác biệt giữa Checked/Unchecked exceptions
  • Sử dụng try-catch-finally và try-with-resources hợp lý
  • Tạo custom exceptions cho các tình huống nghiệp vụ cụ thể
  • Áp dụng exception translation và chaining để cải thiện abstraction
  • Luôn log exceptions đầy đủ và cung cấp thông tin lỗi hữu ích
  • Tuân thủ best practices để code trở nên robust và maintainable
