Giới thiệu Spring Framework

Spring Framework là một trong những framework mạnh mẽ và phổ biến nhất trong hệ sinh thái Java. Nó giúp các lập trình viên xây dựng ứng dụng linh hoạt, dễ bảo trì, và mở rộng nhanh chóng nhờ triết lý "Inversion of Control" (IoC) và "Dependency Injection" (DI).

Nếu bạn từng thấy việc khởi tạo, cấu hình và quản lý các đối tượng trong dự án Java trở nên phức tạp, Spring chính là giải pháp tối ưu hóa mọi thứ đó. Bài viết này sẽ hướng dẫn bạn hiểu các khái niệm cốt lõi của Spring, cách Spring Boot đơn giản hóa phát triển ứng dụng, và cách xây dựng RESTful API nhanh chóng chỉ trong vài dòng code.

Lịch sử và sự phát triển của Spring

Spring Framework được Rod Johnson giới thiệu lần đầu vào năm 2003 trong cuốn sách "Expert One-on-One J2EE Design and Development". Ban đầu, Spring được tạo ra để giải quyết những phức tạp của Enterprise JavaBeans (EJB) và cung cấp một cách tiếp cận nhẹ nhàng hơn để phát triển ứng dụng doanh nghiệp.

Qua nhiều năm, Spring đã phát triển thành một hệ sinh thái toàn diện với nhiều dự án con:

  • Spring Framework Core - Nền tảng chính với DI và IoC
  • Spring Boot - Giúp tạo ứng dụng Spring độc lập một cách nhanh chóng
  • Spring Data - Đơn giản hóa truy cập dữ liệu từ các nguồn khác nhau
  • Spring Security - Framework bảo mật mạnh mẽ
  • Spring Cloud - Công cụ cho kiến trúc microservices

Dependency Injection (DI) – Trái tim của Spring

Dependency Injection là cơ chế cho phép Spring tự động cung cấp (inject) các đối tượng phụ thuộc vào class khác, giúp giảm sự gắn kết (coupling) giữa các module.

Thay vì:

public class UserService {
  private UserRepository userRepository = new UserRepository();
}

Bạn có thể viết:

@Service
public class UserService {
  private final UserRepository userRepository;

  @Autowired
  public UserService(UserRepository userRepository) {
    this.userRepository = userRepository;
  }
}

Ở đây, Spring sẽ tự động "tiêm" UserRepository vào UserService mà bạn không cần khởi tạo thủ công. Điều này giúp việc test, bảo trì và thay đổi code dễ dàng hơn rất nhiều.

Các cách triển khai Dependency Injection

Spring hỗ trợ ba cách chính để thực hiện Dependency Injection:

  • Constructor Injection - Khuyến nghị sử dụng
  • Setter Injection - Thông qua phương thức setter
  • Field Injection - Trực tiếp vào trường với @Autowired

Inversion of Control (IoC)

IoC là nguyên lý cốt lõi của Spring, nơi mà việc quản lý vòng đời của các đối tượng (bean) được chuyển giao cho framework.

Tất cả các bean được Spring quản lý nằm trong IoC Container, và được khai báo bằng annotation (@Component, @Service, @Repository) hoặc trong file cấu hình XML.

Spring IoC Container

Spring có hai loại container chính:

  • BeanFactory - Container cơ bản, cung cấp cấu hình cơ bản
  • ApplicationContext - Container nâng cao, thêm các tính năng doanh nghiệp

Spring Boot – Cách nhanh nhất để bắt đầu

Spring Boot ra đời để đơn giản hóa việc khởi tạo và cấu hình ứng dụng Spring truyền thống. Không cần cấu hình XML phức tạp — chỉ cần vài annotation, bạn có thể chạy ứng dụng backend trong vài giây. 

@SpringBootApplication
public class DemoApplication {
  public static void main(String[] args) {
    SpringApplication.run(DemoApplication.class, args);
  }
}

Khi chạy, Spring Boot tự động:

  • Tạo IoC Container và quản lý bean
  • Cấu hình server (Tomcat) mặc định
  • Quét các component trong project
  • Hỗ trợ build REST API, JPA, Security chỉ với một vài dependency

Spring Boot Starters

Spring Boot cung cấp các "starter" để đơn giản hóa việc thêm dependencies:

  • spring-boot-starter-web - Cho ứng dụng web
  • spring-boot-starter-data-jpa - Cho Spring Data JPA
  • spring-boot-starter-security - Cho Spring Security
  • spring-boot-starter-test - Cho testing

🚀 Spring Boot Auto-Configuration

Spring Boot tự động cấu hình dựa trên các dependency trong classpath. Thêm spring-boot-starter-web sẽ tự động cấu hình Tomcat và Spring MVC.

Spring MVC Framework

Spring MVC (Model-View-Controller) là framework web mạnh mẽ của Spring, cung cấp kiến trúc rõ ràng để phát triển ứng dụng web.

Các thành phần chính của Spring MVC:

  • DispatcherServlet - Front controller, điều phối tất cả request
  • Controller - Xử lý request và trả về response
  • View Resolver - Xác định view để hiển thị
  • Model - Chứa dữ liệu để truyền đến view
@Controller
@RequestMapping("/products")
public class ProductController {
  
  @Autowired
  private ProductService productService;
  
  @GetMapping
  public String listProducts(Model model) {
    model.addAttribute("products", productService.findAll());
    return "products/list";
  }
  
  @GetMapping("/{id}")
  public String getProduct(@PathVariable Long id, Model model) {
    model.addAttribute("product", productService.findById(id));
    return "products/detail";
  }
}

Xây dựng RESTful Web Services với Spring Boot

Một trong những tính năng mạnh mẽ nhất của Spring Boot là xây dựng REST API cực nhanh. Dưới đây là ví dụ đơn giản: 

@RestController
@RequestMapping("/api")
public class HelloController {

  @GetMapping("/hello")
  public String sayHello() {
    return "Xin chào từ Spring Boot!";
  }
}

Chạy ứng dụng và truy cập http://localhost:8080/api/hello, bạn sẽ thấy kết quả trả về từ API mà không cần thêm bất kỳ cấu hình server nào.

Ví dụ REST API hoàn chỉnh với CRUD

@RestController
@RequestMapping("/api/users")
public class UserController {
  
  @Autowired
  private UserService userService;
  
  @GetMapping
  public List getAllUsers() {
    return userService.findAll();
  }
  
  @GetMapping("/{id}")
  public ResponseEntity getUserById(@PathVariable Long id) {
    User user = userService.findById(id);
    return ResponseEntity.ok(user);
  }
  
  @PostMapping
  public ResponseEntity createUser(@RequestBody User user) {
    User savedUser = userService.save(user);
    return new ResponseEntity<>(savedUser, HttpStatus.CREATED);
  }
  
  @PutMapping("/{id}")
  public ResponseEntity updateUser(@PathVariable Long id, @RequestBody User user) {
    User updatedUser = userService.update(id, user);
    return ResponseEntity.ok(updatedUser);
  }
  
  @DeleteMapping("/{id}")
  public ResponseEntity deleteUser(@PathVariable Long id) {
    userService.delete(id);
    return ResponseEntity.noContent().build();
  }
}

"Spring Boot biến việc viết API trở nên đơn giản như viết một dòng println()."

Spring và tầng dữ liệu (Spring Data JPA)

Với module Spring Data JPA, bạn có thể tương tác với cơ sở dữ liệu mà không cần viết câu SQL dài dòng. Chỉ cần định nghĩa interface: 

public interface UserRepository extends JpaRepository {
  List findByName(String name);
  List findByEmailContaining(String email);
  Optional findByUsername(String username);
  List findByAgeGreaterThan(int age);
}

Spring Boot sẽ tự động tạo các truy vấn tương ứng nhờ vào tên phương thức — cực kỳ tiện lợi và tiết kiệm thời gian.

Custom Query với @Query

Khi cần truy vấn phức tạp hơn, bạn có thể sử dụng annotation @Query:

public interface UserRepository extends JpaRepository {
  
  @Query("SELECT u FROM User u WHERE u.status = :status AND u.createdDate > :date")
  List findActiveUsersAfterDate(@Param("status") String status, 
                                      @Param("date") LocalDateTime date);
  
  @Query(value = "SELECT * FROM users WHERE email LIKE %:domain", nativeQuery = true)
  List findUsersByEmailDomain(@Param("domain") String domain);
}

Spring Security - Bảo mật ứng dụng

Spring Security là framework bảo mật mạnh mẽ cung cấp xác thực (authentication) và ủy quyền (authorization) cho ứng dụng Spring.

Cấu hình Spring Security cơ bản

@Configuration
@EnableWebSecurity
public class SecurityConfig {
  
  @Bean
  public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
    http
      .authorizeHttpRequests(authz -> authz
        .requestMatchers("/public/**").permitAll()
        .requestMatchers("/admin/**").hasRole("ADMIN")
        .anyRequest().authenticated()
      )
      .formLogin(form -> form
        .loginPage("/login")
        .permitAll()
      )
      .logout(logout -> logout
        .permitAll()
      );
    
    return http.build();
  }
  
  @Bean
  public UserDetailsService userDetailsService() {
    UserDetails user = User.withDefaultPasswordEncoder()
      .username("user")
      .password("password")
      .roles("USER")
      .build();
    
    UserDetails admin = User.withDefaultPasswordEncoder()
      .username("admin")
      .password("admin")
      .roles("ADMIN")
      .build();
    
    return new InMemoryUserDetailsManager(user, admin);
  }
}

Spring Cloud & Microservices

Spring Cloud cung cấp các công cụ để xây dựng các mẫu phổ biến trong hệ thống phân tán và microservices.

Các thành phần chính của Spring Cloud:

  • Spring Cloud Config - Quản lý cấu hình tập trung
  • Spring Cloud Netflix Eureka - Service discovery
  • Spring Cloud Gateway - API Gateway
  • Spring Cloud Circuit Breaker - Xử lý lỗi và khả năng chịu lỗi
  • Spring Cloud Sleuth - Distributed tracing

Best Practices khi phát triển với Spring

Để phát triển ứng dụng Spring hiệu quả, hãy tuân thủ các best practices sau:

1. Sử dụng Constructor Injection

@Service
public class UserService {
  private final UserRepository userRepository;
  private final EmailService emailService;
  
  public UserService(UserRepository userRepository, EmailService emailService) {
    this.userRepository = userRepository;
    this.emailService = emailService;
  }
}

2. Sử dụng @ConfigurationProperties cho cấu hình

@Configuration
@ConfigurationProperties(prefix = "app")
public class AppConfig {
  private String name;
  private String version;
  private Database database;
  
  // getters and setters
  
  public static class Database {
    private String url;
    private String username;
    private String password;
    
    // getters and setters
  }
}

3. Xử lý Exception với @ControllerAdvice

@ControllerAdvice
public class GlobalExceptionHandler {
  
  @ExceptionHandler(UserNotFoundException.class)
  public ResponseEntity handleUserNotFound(UserNotFoundException ex) {
    ErrorResponse error = new ErrorResponse("USER_NOT_FOUND", ex.getMessage());
    return new ResponseEntity<>(error, HttpStatus.NOT_FOUND);
  }
  
  @ExceptionHandler(Exception.class)
  public ResponseEntity handleGenericException(Exception ex) {
    ErrorResponse error = new ErrorResponse("INTERNAL_ERROR", "Đã xảy ra lỗi hệ thống");
    return new ResponseEntity<>(error, HttpStatus.INTERNAL_SERVER_ERROR);
  }
}

Tổng kết

Spring Framework là lựa chọn hàng đầu cho các lập trình viên Java khi xây dựng ứng dụng web, microservice hoặc hệ thống doanh nghiệp. Việc hiểu và áp dụng Dependency Injection, IoC Container và Spring Boot sẽ giúp bạn tiết kiệm hàng chục giờ cấu hình và phát triển.

Spring không chỉ là một framework mà là một hệ sinh thái toàn diện giúp giải quyết hầu hết các vấn đề trong phát triển ứng dụng doanh nghiệp. Từ ứng dụng web đơn giản đến hệ thống microservices phức tạp, Spring đều cung cấp các công cụ phù hợp.

👉 Hãy bắt đầu với một project Spring Boot nhỏ — tạo một REST API đơn giản, thêm tầng dữ liệu JPA — bạn sẽ hiểu ngay vì sao Spring được gọi là "xương sống của Java backend".

"Học Spring là bước chuyển mình từ lập trình Java cơ bản sang xây dựng hệ thống chuyên nghiệp."
